# 3장 패러다임 개요

패러다임에는 구조적 프로그래밍, 객체지향 프로그래밍, 함수형 프로그래밍 세 가지가 있다.

## 구조적 프로그래밍
- 최초로 적용된 패러다임
- 무분별한 점프(goto 문장)는 프로그램 구조에 해롭다는 사실 제시
- 이러한 점프들을 if/then/else와 do/while/until과 같이 더 익숙한 구조로 대체함.
- 요약하면 구조적 프로그래밍은 제어흐름의 직접적인 전환에 대해 규칙을 부과하는 것

## 객체 지향 프로그래밍
- 요한 달과 크리스텐 니가드 두 프로그래머가 알골(ALGOL) 언어의 함수 호출 스택 프레임을 힙으로 옮기면, 함수 호출이 반환된 이후에도 함수에서 선언된 지역 변수가 오랫동안 유지될 수 있음을 발견함.
- 이러한 함수가 클래스의 생성자가 되었고, 지역 변수는 인스턴스 변수, 중첩 함수는 메서드가 됨
- 함수 포인터를 특정 규칙에 따라 사용하는 과정으로 다형성이 등장하게 됨
- 요약하면 객체 지향 프로그래밍은 제어흐름의 간접적인 전환에 대해 규칙을 부과하는 것

## 함수형 프로그래밍
- 함수형 프로그래밍은 수학적 문제를 해결하는 과정에서 람다 계산법을 이에 영향을 받아 만들어짐
- 람다 계산법의 기초가 되는 개념이 바로 불변성, 심볼의 값이 변경되지 않는다는 것, 이는 함수형 언어에는 할당문이 전혀 없다는 뜻
- 이에 따라 대다수의 함수형 언어가 변수 값을 변경할 수 있는 방법을 제공하기는 하지만, 굉장히 까다로운 조건 아래에서만 가능함
- 요약하면 함수형 프로그래밍은 할당문에 대해 규칙을 부과하는 것

## 결론
- 각 패러다임은 프로그래머에게서 권한을 박탈하는 것, 새로운 권한을 부여하지 않는다.
- 즉, 패러다임은 무엇을 해야할 지를 말하기보다는 무엇을 해서는 안 되는지를 말해준다.
- 이러한 패러다임의 역사로부터 얻을 수 있는 교훈은 아키텍쳐와 관계가 깊다.
- 우리는 아키텍쳐 경계를 넘나들기 위한 메커니즘으로 다형성을 이용하고, 함수형 프로그래밍을 이용하여 데이터의 위치와 접근 방법에 대해 규칙을 부과하고, 모듈의 기반 알고리즘으로 구조적 프로그래밍을 사용함
- 세 가지 패러다임과 아키텍처의 세 가지 큰 관심사(함수, 컴포넌트 분리, 데이터 관리) 가 어떻게 서로 연관되는지를 주목 하는것이 중요하다.
